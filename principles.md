# Principles — 触发式行为准则

> 触发条件匹配时，自动执行对应原则要求的行为。
> 原则来源：用户的直接反馈 + 重复出现的问题。

## 生命周期

```
新建 Principle
    ↓
被触发
    ↓
用户验证"有效/有帮助" → verified（一次确认即可）
    ↓
或者：多次触发表现正确 → verifying → verified（2次触发）
    ↓
用户明确说"不对" → degraded → 降级为 incident
```

**验证信号（二选一即为 verified）：**
- 用户明确说"有效/有帮助/这就是原则"
- 同一 principle 被正确触发 2 次以上

## 降级规则

- 用户说"这个原则不对" → 立即降级
- 降级时：`status: degraded`，原因写入对应 incident
- 降级后：留在文件末尾作为"已失效原则"参考

## AI视频制作

### [P-002] 即梦提示词顺序规范
- **触发条件**: 为即梦AI生成首帧/视频提示词
- **原则**: 严格按顺序执行——1.镜头机位（景别+角度+运镜）2.动作执行 3.物理变化（视觉反馈）
- **禁止**: 先写色调/质感、先写形容词、顺序混乱
- **禁止**: 使用 HEX 颜色代码（#8B7355）—— 即梦会把 HEX 识别成文字
- **来源**: 2026-03-28 用户反馈"提示词执行是按顺序的"
- **状态**: verifying

### [P-003] 首帧生成提示词用纯文本格式
- **触发条件**: 生成供用户复制的提示词文件
- **原则**: 不用 Markdown 格式，用户直接复制使用
- **来源**: 2026-03-28 用户反馈"不要使用 markdown 格式，我不太会复制"
- **状态**: verifying

### [P-004] 先资产后分镜，禁止跳过资产直接生成视频
- **触发条件**: AI视频项目启动或进入制作流程
- **原则**: 必须按顺序执行：资产建模(Reference) → 分镜脚本 → 首帧图 → 视频
- **禁止**: 没有 Reference 就生成视频
- **禁止**: 不同分镜用不同的人物形象
- **来源**: 2026-03-26 雨夜决项目教训
- **状态**: new

## 知识管理

### [P-005] 反馈升级为原则的标准
- **触发条件**: 用户纠正了一个错误或给了一个明确的指示
- **判断标准**:
  - 如果这个错误可能重复发生 → 写成 principle
  - 如果只发生一次 → 只记录到 incidents
  - 存疑时 → 问用户"这要写成 principle 吗？"
- **来源**: 2026-03-28 关于自我进化的讨论
- **状态**: verified

### [P-006] 原则必须有触发条件
- **触发条件**: 写新原则时
- **原则**: 每条原则必须声明 `触发条件` —— 明确说明"在什么情况下这条原则应该被调用"
- **禁止**: 写没有触发条件的原则（等于无效知识）
- **来源**: 2026-03-28 框架设计
- **状态**: verified
