# Principles — 触发式行为准则

> 触发条件匹配时，自动执行对应原则要求的行为。
> 原则来源：用户的直接反馈 + 重复出现的问题。

## 生命周期

```
新建 Principle
    ↓
被触发
    ↓
用户验证"有效/有帮助" → verified（一次确认即可）
    ↓
或者：多次触发表现正确 → verifying → verified（2次触发）
    ↓
用户明确说"不对" → degraded → 降级为 incident
```

**验证信号（二选一即为 verified）：**
- 用户明确说"有效/有帮助/这就是原则"
- 同一 principle 被正确触发 2 次以上

## 降级规则

- 用户说"这个原则不对" → 立即降级
- 降级时：`status: degraded`，原因写入对应 incident
- 降级后：留在文件末尾作为"已失效原则"参考

## 框架原则

### [P-005] 反馈升级为原则的标准
- **触发条件**: 用户纠正了一个错误或给了一个明确的指示
- **判断标准**:
  - 如果这个错误可能重复发生 → 写成 principle
  - 如果只发生一次 → 只记录到 incidents
  - 存疑时 → 问用户"这要写成 principle 吗？"
- **来源**: 2026-03-28 关于自我进化的讨论
- **状态**: verified

### [P-006] 原则必须有触发条件
- **触发条件**: 写新原则时
- **原则**: 每条原则必须声明 `触发条件` —— 明确说明"在什么情况下这条原则应该被调用"
- **禁止**: 写没有触发条件的原则（等于无效知识）
- **来源**: 2026-03-28 框架设计
- **状态**: verified

## AI视频制作

### [P-004] 先资产后分镜，禁止跳过
- **触发条件**: AI视频项目启动或进入制作流程
- **原则**: 必须按顺序执行：资产建模(Reference) → 分镜脚本 → 首帧图 → 视频
- **禁止**: 没有 Reference 就生成视频
- **禁止**: 不同分镜用不同的人物形象
- **来源**: 2026-03-26 雨夜决项目教训
- **状态**: new

### [P-009] 项目入口原则
- **触发条件**: 收到任何项目
- **原则**:
  1. 先识别项目类型（雨夜决=武侠AI视频，大漠孤影=沙漠场景，超现实主义=新类型等）
  2. 加载对应知识模块（按项目类型匹配）
  3. 通盘考虑（项目级视角，不拆成孤立任务）
- **新项目类型处理**: 现有知识地图打底 → 使用中补全 → 项目完成时复盘提炼独立模块
- **禁止**: 等有了完整知识再做
- **来源**: 2026-03-28 项目入口原则讨论
- **状态**: new

### [P-011] 知识库结构性进化原则
- **触发**: 完成AI视频任务后、用户反馈"不行"时、发现新技巧时
- **核心问题**: 这是"单一问题"还是"结构性问题"？

| 类型 | 判断标准 | 处理方式 |
|------|---------|---------|
| **单一问题** | 具体某句词/某个案例不好 | 治标：记录到 incidents.md + 对应已有模块 |
| **结构性问题** | 现有分类/公式/模块有缺陷 | 治本：调整知识地图分类/新增模块/修订公式 |

- **单一问题处理**：
  1. 记录到 incidents.md（分析根因）
  2. 追加到对应已有模块（如 Master_Prompts.md 的破坏性词列表）
  3. 不新增分类

- **结构性问题处理**：
  1. 判断：哪个知识模块的哪层结构出了问题？
  2. 调整：分类/公式/模块本身，不是记录单一问题
  3. 更新知识地图的决策树

- **禁止**：
  - 只记录单一问题而不思考结构性原因
  - 发现结构性缺陷但只记录不调整
  - 调整后不更新知识地图

- **进化节奏**: 使用中自然进化，质量是"用出来"的不是"审出来的"

## 系统维护

### [P-008] Incidents 归档规则
- **触发条件**: 心跳检查时
- **原则**:
  - 已关闭 + 创建时间超过 15 天 → 移动到 `incidents/YYYY-MM.md`
  - 降级（degraded）的 incidents → 永久保留，不归档
  - 归档不影响 principles 的触发（归档的是 incidents，不是 principles）
- **来源**: 2026-03-28 用户要求"需要长时间的验证，查看 token 消耗程度"
- **状态**: new
